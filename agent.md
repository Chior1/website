# Agent 工作说明

本文件用于告诉后续开发者或 AI 助手：这个项目应该怎么继续做。

## 用户特征

用户是计算机小白，不熟悉专业技术概念。

和用户沟通时必须：

- 尽量使用中文。
- 尽量使用大白话。
- 遇到专业词要解释。
- 不要默认用户知道什么是前端、后端、接口、组件、部署、依赖。
- 每次说明都要讲清楚「这件事有什么用」。

## 开发前必须阅读

每次开始开发前，先阅读这些文件：

1. `开发日志.md`
2. `docs/开发需求.md`
3. `docs/技术方案.md`
4. `docs/UI设计规范.md`
5. `docs/执行步骤.md`
6. `docs/术语解释.md`

## 工作规则

- 项目必须分阶段推进。
- 每个阶段完成后必须停止，等待用户审核。
- 用户审核通过前，不进入下一阶段。
- 每个阶段结束前必须更新 `开发日志.md`。
- 不要一次性把所有功能做完。
- 不要擅自接入付费服务。
- 不要擅自处理真实用户隐私文件。

## 当前项目方向

项目是一个音视频文字处理网站。

主要功能：

- 录音生成会议纪要。
- 视频生成可编辑字幕。

第一版目标：

- 先做可点击原型。
- 先使用假数据。
- 先确认页面结构和用户流程。

## 技术方向

计划使用：

- Next.js。
- React。
- TypeScript。

注意：

- 阶段 0 不创建 Next.js 项目。
- 阶段 1 才开始创建页面项目。
- 阶段 3 前才讨论真实 AI 服务。

## UI 方向

页面颜色偏白色。

整体风格：

- 清爽。
- 专业。
- 工具型。
- 好阅读。
- 好操作。

## 阶段暂停要求

每个阶段完成后，最终回复必须明确写：

- 本阶段已经完成。
- 已停止。
- 等待用户审核。
- 不会继续下一阶段，除非用户确认。
