### 阶段 1

- 创建 Next.js 项目基础结构。
- 创建首页，包含「录音生成纪要」和「视频生成字幕」两个主入口。
- 创建工作台，包含项目列表、处理中队列、最近编辑入口和导出记录。
- 创建新建任务页，用假数据模拟选择音频或视频任务。
- 创建音频纪要页，展示逐字稿、说话人、会议纪要、发言人占比和会议时间轴。
- 创建视频字幕页，展示视频预览占位、字幕列表、时间码、搜索替换入口和导出入口。
- 创建设置与隐私页，提前说明文件删除、隐私和导出格式。
- 使用白色系 UI，整体保持清爽、工具型、可读性优先。
- 使用假数据模拟完整流程，没有接入真实 AI 服务。
- 安装基础依赖并生成 `package-lock.json`。
- 通过类型检查、生产构建和安全审计。
- 根据用户审核意见，将品牌名改为「听墨」。
- 在新建任务页补充常见音频和视频格式说明。

### 阶段 2

- 音频纪要页支持导出当前编辑内容为 `.txt` 示例纪要文件。

- 视频字幕页支持删除单条字幕。
- 视频字幕页支持搜索替换字幕文字。
- 视频字幕页支持导出 `.srt`、`.vtt`、`.txt` 示例字幕文件。
- 设置页支持删除确认演示。
- 设置页支持展开查看导出格式说明。
- 通过类型检查、生产构建和安全审计。

## 下一步计划

等网站功能整体完成后，如果用户确认使用 OpenAI 作为第一版真实转写服务，最后接入阶段可以做：

- 增加 `.env.example`，说明 `OPENAI_API_KEY` 怎么配置。
- 安装 OpenAI SDK。
- 新增一个只处理音频的后端接口。
- 没有 API Key 时显示友好提示。
- 上传真实音频后返回转写文字。
- 限制文件类型和大小。
- 不长期保存原始音频文件。

## 重要决定

- 项目按阶段推进，不一次性全部完成。
- 每个阶段完成后必须停下，交给用户审核。
- 第一版先做可点击原型，不直接接真实 AI 转写。
- 所有说明尽量使用中文和大白话。
- 用户是计算机小白，不能默认用户理解专业概念。
- 阶段 1 使用 Next.js、React、TypeScript、普通 CSS 和假数据。
- 阶段 2 只做本地浏览器交互，不保存到真实后台。
- 阶段 3 只完成接入前评审，不直接调用付费服务。
- 阶段 3 已预留接口，但接口只返回占位提示，不调用 AI。
- 真实语音识别、说话人区分、视频处理和文件存储要单独确认后再实现。
