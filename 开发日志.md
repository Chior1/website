## 已完成事项

### 阶段 0

- 创建项目开发日志。
- 创建 `docs` 文件夹。
- 创建开发需求、技术方案、UI 设计规范、执行步骤、术语解释等标准文件。
- 创建 `agent.md`，记录后续工作说明和用户沟通方式。

### 阶段 1

- 创建 Next.js 项目基础结构。
- 创建首页，包含「录音生成纪要」和「视频生成字幕」两个主入口。
- 创建工作台，包含项目列表、处理中队列、最近编辑入口和导出记录。
- 创建新建任务页，用假数据模拟选择音频或视频任务。
- 创建音频纪要页，展示逐字稿、说话人、会议纪要、发言人占比和会议时间轴。
- 创建视频字幕页，展示视频预览占位、字幕列表、时间码、搜索替换入口和导出入口。
- 创建设置与隐私页，提前说明文件删除、隐私和导出格式。
- 使用白色系 UI，整体保持清爽、工具型、可读性优先。
- 使用假数据模拟完整流程，没有接入真实 AI 服务。
- 安装基础依赖并生成 `package-lock.json`。
- 通过类型检查、生产构建和安全审计。
- 根据用户审核意见，将品牌名改为「听墨」。
- 在新建任务页补充常见音频和视频格式说明。

### 阶段 2

- 音频纪要页支持本地编辑逐字稿文字。
- 音频纪要页支持修改说话人名称。
- 音频纪要页支持编辑会议摘要、关键结论和待办事项。
- 音频纪要页支持导出当前编辑内容为 `.txt` 示例纪要文件。
- 视频字幕页支持编辑字幕开始时间、结束时间和字幕文字。
- 视频字幕页支持拆分字幕块。
- 视频字幕页支持合并下一条字幕。
- 视频字幕页支持删除单条字幕。
- 视频字幕页支持搜索替换字幕文字。
- 视频字幕页支持导出 `.srt`、`.vtt`、`.txt` 示例字幕文件。
- 设置页支持删除确认演示。
- 设置页支持展开查看导出格式说明。
- 通过类型检查、生产构建和安全审计。
- 页面检查确认音频纪要、视频字幕、设置页的关键入口都能显示。

## 下一步计划

阶段 3 计划方向：

- 和用户确认是否接入真实语音转文字服务。
- 和用户确认文件保存在哪里、保存多久、如何删除。
- 和用户确认是否需要说话人区分。
- 和用户确认是否接受可能产生的服务费用。
- 先做一个最小真实音频转写流程。
- 阶段 3 开始前必须先商量技术选择，不直接接付费服务。

## 重要决定

- 项目按阶段推进，不一次性全部完成。
- 每个阶段完成后必须停下，交给用户审核。
- 第一版先做可点击原型，不直接接真实 AI 转写。
- 所有说明尽量使用中文和大白话。
- 用户是计算机小白，不能默认用户理解专业概念。
- 阶段 1 使用 Next.js、React、TypeScript、普通 CSS 和假数据。
- 阶段 2 只做本地浏览器交互，不保存到真实后台。
- 真实语音识别、说话人区分、视频处理和文件存储要到阶段 3 前再单独确认。
