# UI 设计规范

## 总体方向

网站 UI 使用白色系，整体感觉要清爽、专业、容易读。

关键词：

- 白色背景。
- 浅灰分隔。
- 蓝色或青色作为重点按钮颜色。
- 页面留白充足。
- 工具型界面，不做花哨装饰。
- 重要信息清楚，不靠用户猜。

## 设计原则

1. 先让用户知道下一步该点哪里。
2. 上传、处理中、编辑、导出这些状态要清楚。
3. 文字不要太小。
4. 卡片和表格要整齐。
5. 按钮颜色要有主次。
6. 不使用大面积深色背景。
7. 不使用夸张渐变。
8. 不做看起来漂亮但影响操作的动画。

## 页面结构

### 首页

首页包含：

- 顶部导航。
- 主标题。
- 两个主要功能入口。
- 简单流程说明。
- 适用场景。
- 隐私说明。

首页第一屏就要让用户看到两个入口：

- 录音生成纪要。
- 视频生成字幕。

### 工作台

工作台包含：

- 左侧菜单。
- 顶部状态区域。
- 项目卡片或项目表格。
- 处理中队列。
- 最近编辑。
- 导出记录。

左侧菜单包含：

- 项目。
- 音频纪要。
- 视频字幕。
- 导出中心。
- 团队空间。
- 设置。

### 音频纪要编辑页

音频纪要页建议采用左右结构：

- 左侧：逐字稿和说话人。
- 右侧：会议纪要、待办事项、关键结论。
- 上方：项目状态、导出按钮。
- 下方或侧边：时间轴和发言人占比。

### 视频字幕编辑页

视频字幕页建议采用编辑器结构：

- 左侧：视频预览。
- 下方：字幕时间轴或波形占位。
- 右侧：字幕列表。
- 顶部：搜索替换、导出按钮。

## 组件规则

大白话：组件就是页面里可以重复使用的小部件，比如按钮、卡片、输入框。

常用组件包括：

- 按钮。
- 输入框。
- 文件上传区。
- 项目卡片。
- 状态标签。
- 进度条。
- 字幕编辑行。
- 纪要内容块。

这些组件要保持统一：

- 同一种按钮长得一样。
- 同一种卡片间距一样。
- 同一种状态颜色一样。

## 移动端规则

手机端先保证能看、能简单操作。

第一版手机端要求：

- 首页能正常阅读。
- 工作台项目列表能查看。
- 音频纪要和视频字幕结果能查看。
- 简单编辑可以做。

手机端暂时不强求复杂字幕时间轴编辑，因为手机屏幕太小，复杂操作体验不好。

## 可视化规则

项目里会用少量可视化，不做复杂图表。

可视化包括：

- 发言人时间占比。
- 会议时间轴。
- 处理进度。
- 字幕时间位置。

这些图表的目的不是炫酷，而是让用户更快理解内容。
