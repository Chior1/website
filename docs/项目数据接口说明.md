# 项目数据接口说明

本文件说明阶段 13：项目数据接口。

## 当前完成了什么

项目已经新增 Supabase 项目数据接口。

接口包括：

- `GET /api/projects`：读取项目列表。
- `POST /api/projects`：创建项目。
- `GET /api/projects/[id]`：读取单个项目。
- `PATCH /api/projects/[id]`：更新项目状态。
- `DELETE /api/projects/[id]`：删除项目。

大白话：

- 以前项目只存在浏览器里。
- 现在网站已经有了“问后端要项目”和“把新项目交给后端保存”的入口。
- 后端配置好数据库密码后，就可以真正读写 Supabase。

## 页面怎么使用

工作台会优先读取 Supabase 项目。

如果本地还没有配置数据库连接，页面会退回浏览器本地演示数据。

新建任务会优先创建 Supabase 项目。

如果后端不可用，新建任务会退回本地演示项目。

项目详情页会优先读取 Supabase 单个项目。

如果后端不可用，会尝试读取本地演示项目。

## 当前边界

- 不接入 AI。
- 不上传真实音视频。
- 不做登录注册。
- 草稿仍然保存在浏览器本地。
- 导出记录仍然保存在浏览器本地。
- 如果没有 `.env` 数据库连接，后端接口会返回友好提示。

## 本地需要配置什么

要让接口真正连接 Supabase，需要在本地创建 `.env` 文件。

`.env` 里需要：

- `DATABASE_URL`
- `DIRECT_URL`

注意：

- 这两个地址里包含数据库密码。
- 不能写进聊天。
- 不能提交到 Git。
- 项目已经用 `.gitignore` 忽略 `.env`。
