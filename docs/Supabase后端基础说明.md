# Supabase 后端基础说明

本文件说明阶段 12：为网站建立 Supabase 后端基础。

## 当前完成了什么

项目已经连接到 Supabase MCP，并在 Supabase 项目里创建了第一版数据表。

Supabase 项目地址：

- `https://rhkygoxxqoemioizlkja.supabase.co`

已经创建的数据表：

- `projects`：保存项目，比如录音纪要项目或视频字幕项目。
- `project_drafts`：保存每个项目的编辑草稿。
- `export_records`：保存导出记录。

大白话：

- 以前项目数据只存在浏览器里。
- 现在已经准备好云端数据库。
- 下一阶段可以让网站页面真正读写这些云端数据。

## 本阶段没有做什么

- 没有接入 AI。
- 没有上传真实音频或视频。
- 没有做登录注册。
- 没有把页面切换到云端数据。
- 没有把数据库密码写进代码。

## 本地代码新增了什么

- Prisma 配置：告诉项目数据库有哪些表。
- 数据库连接工具：后续接口可以统一使用。
- `.env.example`：告诉开发者需要配置哪些环境变量。
- 本地迁移文件：记录本次 Supabase 表结构。

大白话：

- Prisma 是网站和数据库之间的翻译员。
- `.env` 是本机保存密码的地方，不能提交到 Git。
- `.env.example` 只放示例，不放真实密码。

## 安全说明

三张表都已经开启 RLS。

大白话：

- RLS 是 Supabase 的表级安全开关。
- 当前没有开放前端直接读写这些表。
- 后续建议通过 Next.js 后端接口读写数据库，不让浏览器直接改核心数据。

Supabase 安全检查当前还有 3 条信息级提示：三张表开启了 RLS，但还没有配置访问策略。

这符合当前阶段预期，因为本阶段不让浏览器直接访问这些表。后续如果要做登录和前端直连，再单独设计访问策略。

Supabase 性能检查当前还有 1 条信息级提示：`export_records_project_id_idx` 索引还没有被使用。

这也符合当前阶段预期，因为表刚创建，还没有真实数据和查询。

## 后续怎么继续

下一阶段建议做“项目数据接口”：

- 新增创建项目接口。
- 新增读取项目列表接口。
