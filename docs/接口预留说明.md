# 接口预留说明

本文件说明：项目先保留未来接入 AI 的接口位置，但现在不真正调用 AI。

## 当前决定

先不接入 AI。

原因：

- 现在还在完善产品页面和本地功能。
- 真实 AI 可能产生费用。
- 真实文件涉及隐私。
- 等网站主要功能完成后，再最后接入 AI，更稳妥。

## 已预留的接口

接口地址：

- `GET /api/transcriptions/audio`
- `POST /api/transcriptions/audio`

大白话解释：

- `GET` 用来看接口说明。
- `POST` 是未来真正上传音频并转写的入口。
- 现在 `POST` 不会调用 AI，只会返回“AI 暂未接入”的提示。

## 未来上传字段

未来真实接入时，前端会用表单方式提交：

- `file`：音频文件。
- `language`：语言，可选，比如 `zh` 表示中文。
- `diarization`：是否需要说话人区分，可选。

## 未来返回格式

未来成功时，接口大概返回：

```json
{
  "status": "completed",
  "provider": "openai",
  "model": "gpt-4o-mini-transcribe",
  "message": "转写完成",
  "text": "这里是转写出来的文字",
  "segments": [
    {
      "start": "00:00:01",
      "end": "00:00:04",
      "speaker": "发言人 1",
      "text": "第一句话"
    }
  ]
}
```

## 当前占位返回

现在调用 `POST /api/transcriptions/audio` 会返回：

```json
{
  "status": "reserved_only",
  "provider": "openai",
  "model": "gpt-4o-mini-transcribe",
  "message": "音频转写接口已预留，但 AI 暂未接入。等网站功能完成后，再配置 API Key 并启用真实转写。"
}
```

## 支持格式预留

先按常见音频格式预留：

- `mp3`
- `mp4`
- `mpeg`
- `mpga`
- `m4a`
- `wav`
- `webm`

## 后续真正接入时再做

后续最后接入 AI 时再做：

- 配置 `OPENAI_API_KEY`。
- 安装 OpenAI SDK。
- 限制文件大小。
- 校验文件类型。
- 调用真实转写服务。
- 返回真实文字。
- 失败时给用户大白话提示。
- 明确文件是否保存、保存多久、如何删除。
