# 执行步骤

## 总规则

项目必须分阶段推进。

每个阶段都要做四件事：

1. 明确本阶段要做什么。
2. 完成本阶段开发或文档。
3. 更新 `开发日志.md`。
4. 停下来等待用户审核。

用户没有审核通过前，不进入下一阶段。

## 阶段 0：项目规则和文档框架

目标：

- 创建开发日志。
- 创建 docs 文档。
- 创建 agent.md。
- 写清楚项目规则。

不做：

- 不创建 Next.js 项目。
- 不开发页面。
- 不安装依赖。
- 不接真实 AI。

完成后：

- 停止。
- 等待用户审核。

## 阶段 1：可点击产品原型

目标：

- 创建 Next.js 项目。
- 搭建首页。
- 搭建工作台。
- 搭建音频纪要编辑页。
- 搭建视频字幕编辑页。
- 使用假数据模拟完整流程。

技术：

- Next.js。
- React。
- TypeScript。
- 白色系 UI。

完成后：

- 运行页面检查。
- 更新开发日志。
- 停止。
- 等待用户审核。

## 阶段 2：本地交互增强

目标：

- 让转写稿可以编辑。
- 让发言人名称可以修改。
- 让字幕块可以编辑、拆分、合并。
- 增加搜索替换。
- 增加示例导出。
- 增加删除确认和隐私提示。

完成后：

- 检查编辑和导出是否可用。
- 更新开发日志。
- 停止。
- 等待用户审核。

## 阶段 3：真实音频处理接入前评审

目标：

- 和用户确认真实语音识别服务。
- 和用户确认文件存储方式。
- 和用户确认费用、隐私、速度要求。
- 做最小真实音频转写流程。

注意：

- 进入本阶段前必须单独确认技术选择。
- 不能默认用户已经同意使用某个付费服务。

## 阶段 4：本地项目管理补全

目标：

- 新建任务可以保存到浏览器本地。
- 工作台可以看到新增项目。
- 项目可以标记完成。
- 项目可以删除。
- 可以恢复示例项目。
- 处理中队列和项目状态联动。

注意：

- 本阶段不接入 AI。
- 本阶段不上传真实文件。
- 本阶段不使用真实数据库。
- 当前只是让产品管理流程更完整。

## 最后阶段：真实 AI 接入

目标：

- 在网站主要功能完成后，再接入真实语音转文字。
- 使用已经预留的 `/api/transcriptions/audio` 接口。
- 配置 API Key。
- 限制文件类型和大小。
- 返回真实转写文字。

注意：

- 进入本阶段前必须再次确认费用、隐私和技术选择。
- 不能默认用户已经同意使用付费服务。

## 阶段 5：本地文件选择预览

目标：

- 新建任务页可以选择本地音频或视频文件。
- 页面显示文件名、大小、格式和浏览器识别类型。
- 创建项目时保存这些文件信息。
- 工作台显示项目对应的文件大小和格式。

注意：

- 本阶段不上传文件。
- 本阶段不读取音频或视频内容。
- 本阶段不接入 AI。
- 本阶段只做本地预览。

## 阶段 6：项目详情页闭环

目标：

- 每个本地项目都有自己的详情页。
- 工作台点击“打开”进入对应项目详情页。
- 音频项目进入音频纪要界面。
- 视频项目进入字幕编辑界面。
- 项目不存在时显示友好提示。

注意：

- 本阶段不接入 AI。
- 本阶段仍然使用示例转写稿和示例字幕。
- 本阶段不为每个项目保存独立编辑内容。

## 阶段 7：项目搜索和筛选

目标：

- 工作台支持关键词搜索。
- 工作台支持项目类型筛选。
- 工作台支持项目状态筛选。
- 没有结果时显示友好提示。
- 支持一键清空筛选条件。

注意：

- 本阶段只搜索浏览器本地项目数据。
- 不搜索真实音视频内容。
- 不连接真实数据库。

## 阶段 8：项目草稿独立保存

目标：

- 音频纪要项目可以保存自己的编辑草稿。
- 视频字幕项目可以保存自己的编辑草稿。
- 再次打开同一个项目时恢复草稿。
- 不同项目之间的草稿互不影响。

注意：

- 本阶段只保存到浏览器本地。
- 不上传到服务器。
- 不接入 AI。
- 当前采用手动点击“保存草稿”，不做自动保存。

## 阶段 9：导出记录管理

目标：

- 导出纪要或字幕时写入本地导出记录。
- 工作台显示本地导出记录。
- 支持删除单条导出记录。
- 支持恢复示例导出记录。

注意：

- 本阶段只记录导出行为。
- 不保存导出文件副本。
- 不上传到服务器。
- 不接入真实数据库。

## 阶段 10：本地数据清理和格式校验

目标：

- 新建任务时检查文件格式是否匹配当前任务类型。
- 文件格式不匹配时显示友好提示。
- 文件格式不匹配时阻止创建任务。
- 删除项目时同步清理草稿。
- 删除项目时同步清理导出记录。

注意：

- 本阶段只处理浏览器本地数据。
- 不删除用户电脑里已经下载的文件。
- 不上传文件。
- 不接入 AI。

## 阶段 11：草稿自动保存

目标：

- 音频纪要编辑后自动保存本地草稿。
- 视频字幕编辑后自动保存本地草稿。
- 保留“保存草稿”按钮，方便用户手动保存。
- 页面显示自动保存或手动保存提示。

注意：

- 本阶段只保存到当前浏览器。
- 不上传到服务器。
- 不接入 AI。
- 自动保存有短暂延迟，避免每敲一个字都保存。

## 阶段 12：Supabase 后端基础

目标：

- 安装 Prisma 和 `@prisma/client`。
- 配置 Supabase PostgreSQL 数据库结构。
- 创建项目、草稿、导出记录三张基础表。
- 创建数据库连接工具。
- 创建 `.env.example`，说明本地环境变量怎么配置。

注意：

- 本阶段只建立后端基础。
- 不接入 AI。
- 不上传真实音视频。
- 不做登录注册。
- 不把真实数据库密码写进代码。
- 完成后必须停止，等待用户审核。

## 阶段 13：项目数据接口

目标：

- 新增项目列表接口。
- 新增创建项目接口。
- 新增单个项目读取接口。
- 新增项目状态更新接口。
- 新增项目删除接口。
- 工作台优先读取 Supabase 项目。
- 新建任务优先写入 Supabase。
- 项目详情页优先读取 Supabase 项目。

注意：

- 本阶段只处理项目基础数据。
- 不接入 AI。
- 不上传真实音视频。
- 不做登录注册。
- 草稿和导出记录后端化留到后续阶段。
- 如果没有 `.env` 数据库连接，页面要能退回本地演示。

## 每阶段交付格式

每个阶段结束时，需要告诉用户：

- 完成了什么。
- 哪些文件发生变化。
- 怎么检查。
- 哪些事情还没做。
- 是否已经停止等待审核。
