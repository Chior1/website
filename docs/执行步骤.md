# 执行步骤

## 总规则

项目必须分阶段推进。

每个阶段都要做四件事：

1. 明确本阶段要做什么。
2. 完成本阶段开发或文档。
3. 更新 `开发日志.md`。
4. 停下来等待用户审核。

用户没有审核通过前，不进入下一阶段。

## 阶段 0：项目规则和文档框架

目标：

- 创建开发日志。
- 创建 docs 文档。
- 创建 agent.md。
- 写清楚项目规则。

不做：

- 不创建 Next.js 项目。
- 不开发页面。
- 不安装依赖。
- 不接真实 AI。

完成后：

- 停止。
- 等待用户审核。

## 阶段 1：可点击产品原型

目标：

- 创建 Next.js 项目。
- 搭建首页。
- 搭建工作台。
- 搭建音频纪要编辑页。
- 搭建视频字幕编辑页。
- 使用假数据模拟完整流程。

技术：

- Next.js。
- React。
- TypeScript。
- 白色系 UI。

完成后：

- 运行页面检查。
- 更新开发日志。
- 停止。
- 等待用户审核。

## 阶段 2：本地交互增强

目标：

- 让转写稿可以编辑。
- 让发言人名称可以修改。
- 让字幕块可以编辑、拆分、合并。
- 增加搜索替换。
- 增加示例导出。
- 增加删除确认和隐私提示。

完成后：

- 检查编辑和导出是否可用。
- 更新开发日志。
- 停止。
- 等待用户审核。

## 阶段 3：真实音频处理接入前评审

目标：

- 和用户确认真实语音识别服务。
- 和用户确认文件存储方式。
- 和用户确认费用、隐私、速度要求。
- 做最小真实音频转写流程。

注意：

- 进入本阶段前必须单独确认技术选择。
- 不能默认用户已经同意使用某个付费服务。

## 阶段 4：真实字幕和导出能力

目标：

- 从视频中提取音频。
- 生成真实字幕。
- 导出 `.srt`、`.vtt`、纯文本。

注意：

- 复杂字幕样式、翻译、多轨道编辑不放进第一版。

## 每阶段交付格式

每个阶段结束时，需要告诉用户：

- 完成了什么。
- 哪些文件发生变化。
- 怎么检查。
- 哪些事情还没做。
- 是否已经停止等待审核。
